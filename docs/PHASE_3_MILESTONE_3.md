# Phase 3 Milestone 3: Cross-Artifact Temporal Comparison

Phase 3 Milestone 3 extends temporal analysis from one lawful artifact to governed comparison between multiple lawful artifacts. This milestone should happen only after Milestones 1 and 2 are stable enough that single-artifact temporal semantics are no longer moving targets.

The purpose of this milestone is not visual comparison. It is lawful temporal comparison grounded in explicit observed-set basis, temporal policy, and relational provenance.

## Intent

Milestone 3 makes it possible to ask:

- how two lawful artifacts differ in temporal drift and propagation
- whether one artifact diverges from, converges toward, or counterfeits another temporal structure
- whether topology change is parallel, delayed, or incompatible across artifacts

## Scope

Milestone 3 includes:

- cross-artifact temporal basis alignment
- comparison-horizon normalization rules
- artifact-to-artifact temporal drift comparison
- artifact-to-artifact relation-graph topology comparison
- governed classification of temporal convergence, divergence, and incompatibility

Milestone 3 does not include:

- engrammatic identity comparison
- formation or covenant outcomes
- runtime OE or Sanctuary continuity behavior
- aesthetic phase rendering

## Required Preconditions

Before Milestone 3 begins:

- Milestone 1 slice semantics must be stable
- Milestone 2 state classification and derived force semantics must be stable enough to compare
- reference artifacts must exist with compatible policy and lawful temporal provenance

## Comparison Model

Cross-artifact temporal comparison should require explicit basis alignment:

- compatible observed-set cadence or lawful normalization rule
- compatible horizon meaning
- compatible channel semantics
- lawful universe and relation reference mapping

Primary comparison outputs:

- `Convergent`
- `Divergent`
- `Delayed`
- `Incompatible`
- `FlattenedOrUnsupported`

Supporting metrics:

- drift delta
- topology delta
- derived force delta
- similarity score as a secondary diagnostic only

## Command Surface

Milestone 3 should introduce a governed temporal comparison command rather than overloading `compare-surfaces`.

Preferred public surface:

- `Compare-HOPNGPhaseStacks.ps1`

The command should compare two artifacts and report:

- basis alignment result
- temporal-state compatibility
- drift and topology deltas
- convergence or divergence classification
- Prime-safe vs privileged output posture

## Validation Rules

Milestone 3 must enforce:

- lawful basis alignment before temporal comparison proceeds
- clear failure states for incompatible cadence, horizon, or policy
- no silent normalization that hides semantic mismatch
- Prime-safe cross-artifact output that does not expose protected temporal payloads

## Acceptance Criteria

- two lawful artifacts can be compared temporally under explicit basis rules
- incompatible artifacts fail comparison clearly rather than being coerced
- convergence and divergence classification is deterministic
- Prime-safe temporal comparison output remains lawful and metadata-first

## Non-Goals

- identity-bearing or engrammatic cross-artifact semantics
- commitment or covenant interpretation of convergence
- Sanctuary or OE runtime action based on temporal comparison
