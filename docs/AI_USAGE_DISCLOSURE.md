# AI Usage Disclosure Policy

## Purpose

Lucid Technologies permits responsible AI assistance in this repository, but requires disclosure when AI materially affects repository content.

## Disclosure Rule

Contributors must disclose AI use in a pull request when AI assistance materially contributed to:

- source code
- schemas
- tests
- documentation
- commit text or design rationale that affects technical review
- generated examples or transformed datasets

Minor editorial assistance that does not change technical meaning does not require detailed disclosure, but contributors may still note it.

## Minimum Disclosure

When disclosure is required, include:

- tool or model family used
- what parts of the contribution were AI-assisted
- what human review was performed
- whether any generated output was edited, tested, or independently verified
- whether external data or proprietary prompts were provided to the tool

## Lucid Technologies Rules

- AI-generated output is never accepted without human review
- contributors remain responsible for correctness, provenance, and licensing
- AI output must not be presented as empirical evidence without validation
- contributors must not paste secrets, private keys, restricted data, or protected participant information into external AI systems without authorization
- maintainers may reject contributions with inadequate disclosure or insufficient human verification

## Repository Interpretation

For Holographic Data Tool, disclosure is especially important when AI materially influences:

- trust envelope logic
- signature verification
- artifact validation rules
- visibility boundary behavior
- governance or compliance documentation
