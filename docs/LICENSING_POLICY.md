# Licensing Policy

## Current Repository License Position

Until Lucid Technologies adds an explicit top-level `LICENSE` file, this repository should be treated as **all rights reserved** except where law provides otherwise. Contributors and downstream users must not assume an open-source license exists merely because the repository is visible on GitHub.

## Lucid Technologies Rule

- no contributor may add a new third-party dependency, asset, dataset, schema, or generated artifact without confirming its license terms are compatible with repository use
- no file may be copied from another project without preserving required attribution and license notice obligations
- no contributor may claim the repository is MIT, Apache-2.0, GPL, or any other named license unless a maintainer adds that license explicitly

## Inbound Contribution Rule

Unless maintainers state otherwise in writing:

- contributors retain rights to their contributions
- by submitting a contribution, the contributor attests they have the right to submit it
- the contribution may be reviewed, stored, modified, and distributed by Lucid Technologies under the repository’s then-current licensing terms

## Third-Party Material

Each pull request introducing third-party material must state:

- source location
- copyright or ownership status
- applicable license
- whether attribution or notice files are required
- whether the material is code, model output, documentation, data, or media

## Maintainer Guidance

If Lucid Technologies later adopts an explicit repository license:

- add a top-level `LICENSE` file
- update this document to reference the chosen license
- confirm whether inbound contributions remain governed by DCO, CLA, or both
